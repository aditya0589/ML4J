# Contributing to ML4J

Thank you for your interest in contributing to ML4J! We welcome contributions from the community to help make this library better.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct. Please treat everyone with respect and kindness.

## How to Contribute

### Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub. When reporting a bug, please include:
-   A clear and descriptive title.
-   Steps to reproduce the issue.
-   Expected behavior vs. actual behavior.
-   Environment details (Java version, OS, etc.).

### Submitting Pull Requests

1.  **Fork the Repository**: Create a fork of the ML4J repository to your own GitHub account.
2.  **Create a Branch**: Create a new branch for your feature or bug fix.
    ```bash
    git checkout -b feature/my-new-feature
    ```
3.  **Make Changes**: Implement your changes. Ensure your code follows the project's coding style.
4.  **Test**: Run existing tests and add new ones if applicable.
    ```bash
    mvn test
    ```
5.  **Commit**: Commit your changes with clear and descriptive messages.
    ```bash
    git commit -m "Add feature X"
    ```
6.  **Push**: Push your branch to your fork.
    ```bash
    git push origin feature/my-new-feature
    ```
7.  **Open a Pull Request**: Go to the original ML4J repository and open a Pull Request (PR) from your branch. Provide a detailed description of your changes.

## Coding Standards

-   Follow standard Java naming conventions.
-   Write clean, readable, and documented code.
-   Ensure all new code is covered by unit tests where possible.
